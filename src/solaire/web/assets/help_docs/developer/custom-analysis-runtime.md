# 自定义分析脚本运行机制（开发者）

本文记录 EduAnalysis 的自定义数据处理脚本能力，便于二次开发、排障和扩展。

## 1. 入口与执行路径

- 用户在工作区选择项目 `analysis/` 目录下脚本。
- 后端创建任务并进入脚本执行器。
- 执行器会为本次考试与批次准备运行数据，再在隔离进程中执行脚本。

## 2. 脚本可用能力

脚本执行时会自动注入：

- `get_rawdata()`：返回当前考试/批次的结构化统计数据。
- `get_graph()`：返回与本次分析相关的图谱子集。
- `BaseChart`、`HistogramChart`、`PieChart`：图形对象基类与内置图形类型。

脚本约定：

- 将最终结果写入 `RESULT`。
- `RESULT` 可为图形对象实例，或结构化结果对象。

## 3. 图形对象回传规则

- 若 `RESULT` 具备 `to_payload()` 与 `get_picture()`，执行器按图形对象处理。
- `to_payload()` 用于结构化结果回传。
- `get_picture()` 用于生成图形图片（当前为 SVG 文本）。
- 最终任务输出统一写入 `output`，并包含 `summary`、`tables`、`chart_specs`、`series`、`pictures` 等字段。

## 4. 安全与资源限制

- 执行前进行语法树检查，限制导入范围。
- 运行时采用受限内建函数集合。
- 对执行时长、输出体量、并发数量进行限制。
- 超限会返回明确错误码（如 `timeout`、`resource_exceeded`、`sandbox_violation`）。

## 5. 执行器内省实现说明

当前执行器采用“函数内省”方式生成运行脚本：

- 沙箱主逻辑写成真实 Python 函数。
- 通过 `inspect.getsource()` 在运行时提取源码，写入临时 `runner.py`。
- 白名单与内建函数清单通过运行策略文件传入，不再依赖大段模板替换。

这种方式的好处是：主逻辑可调试、可测试、可重构，避免超长字符串模板带来的维护成本。
