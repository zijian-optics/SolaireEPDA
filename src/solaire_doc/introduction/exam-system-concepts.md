# 组卷系统如何运作（概念说明）

下面用**流程图**说明「考试项目、题库、模板、组卷与导出」的关系。具体操作仍请在「组卷」「题库」「模板」等界面完成；本文**不是**逐步点击教程。

---

## 考试项目里有什么

```mermaid
flowchart TB
  subgraph bag["您打开的「考试项目」文件夹"]
    A[题库\n题目与资源]
    B[模板\n试卷结构与版式]
    C[导出结果\n生成的试卷文件]
  end
  A --> D[日常主要编辑区]
  B --> D
  C --> E[定稿后从这里取 PDF 等]
```

---

## 题库、模板与组卷：谁依赖谁

```mermaid
flowchart LR
  subgraph need["组卷前须具备"]
    Q[题库里有题]
    T[已有一份试卷模板]
  end
  Q --> G[组卷]
  T --> G
  G --> S[得到「选题方案」\n哪几道题进哪一节]
```

没有题目，组卷没有内容；没有模板，就没有「这一场考」的章节与版式骨架。

---

## 从选题到拿到试卷：主流程

```mermaid
flowchart TD
  start([开始组卷]) --> pickT[选定模板]
  pickT --> pickQ[在各小节从题库选题]
  pickQ --> scores[可按小节或逐题设置分值\n可保存草稿或载入历史试卷]
  scores --> plan[形成编排方案]
  plan --> check{检查/校对}
  check -->|有问题| fix[按提示补充或调整]
  fix --> check
  check -->|通过| export[生成试卷文件]
  export --> endNode([导出结果目录中可取用])
```

「检查」会核对模板要求、题目是否存在、资源是否齐全等；**通过后再导出**，可减少白跑一趟。

---

## 与知识图谱、分析、智能助手的关系（可选）

```mermaid
flowchart BT
  main[主线：题库 → 模板 → 组卷 → 检查 → 导出]
  kg[知识图谱\n题目与知识点关联]
  ana[分析\n统计与报表]
  asst[智能助手侧栏\n自然语言协助]
  main -.->|可选| kg
  main -.->|可选| ana
  kg -.-> asst
  ana -.-> asst
  main -.-> asst
```

图谱、分析与智能助手**不改变**上述主线顺序；助手可在各业务页叠加使用，便于整理题目、维护图谱或触发分析等操作。
